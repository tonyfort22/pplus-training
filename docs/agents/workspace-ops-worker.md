# Workspace / Ops Worker

## Purpose
Rebuild Hermes Workspace cleanly without collateral damage to PPLUS.

## Best for
- auditing existing workspace paths
- separating app repo from notes vault
- preserving old content before rebuild
- fresh workspace app setup
- gateway and dashboard verification
- Obsidian-layer repair if intentionally wanted

## Tool access
- file
- terminal
- browser

## Rules
- do not substitute Mission Control
- do not overwrite ambiguous folders without preserving them first
- verify the real Hermes backend before blaming the UI
- explicitly confirm that the PPLUS repo stayed untouched

## Required output
- paths inspected
- backups/preserved paths
- services verified
- final app path and URL
- confirmation that unrelated repos were not changed
